# subF-20 · Provider API: «Увійти через monobank»
**Фаза:** 4 · **Розмір:** M · **Блокери:** апрув заявки subF-2 (може прийти в будь-який момент — тікет готовий до підняття)

## Задачі
- [ ] Імплементація MonoProviderApi (той самий інтерфейс BankProvider з subF-8): підпис X-Sign усіх запитів (X-Time | X-Request-Id | URL, secp256k1), X-Key-Id
- [ ] Корпоративний вебхук: `POST /personal/corp/webhook` (ендпоінт має відповідати 200 ДО реєстрації)
- [ ] Флоу авторизації: `POST /personal/auth/request` (+X-Callback) → acceptUrl → відкриття діплінка mbnk.app з Flutter → юзер підтверджує в застосунку mono → callback/поллінг статусу → збереження tokenRequestId як X-Request-Id
- [ ] Онбординг v2 у Flutter: кнопка «Підключити monobank» замість ручного токена (екран 3 з subF-14 замінюється одним тапом)
- [ ] Міграція: існуючі personal-конекшени продовжують працювати; нові — через provider; статистика по обох
- [ ] Обробка відкликання доступу (401/403) → флоу перепідключення

## Acceptance Criteria
Новий юзер підключає mono за ≤60 сек без ручного токена; обидва типи конекшенів співіснують; бекфіл/вебхуки/детекція працюють ідентично для обох.
