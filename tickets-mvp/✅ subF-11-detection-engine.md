# subF-11 · Рушій детекції підписок (ЯДРО ПРОДУКТУ)
**Фаза:** 2 · **Розмір:** L · **Блокери:** subF-7, subF-9 (можна стартувати на власних даних одразу після першого бекфілу)
**Де живе:** `packages/detection` — чиста бібліотека, тестується офлайн (subF-12)

## Задачі
### Нормалізація мерчантів
- [ ] Пайплайн: lowercase → зрізати процесингові префікси/суфікси (`PAYPAL *`, `GPC *`, `APPLE.COM/BILL`, `WWW.`, локаційні хвости) → трім спецсимволів → canonical_key = norm_name + mcc
- [ ] Матчинг до seed-довідника: точні патерни + fuzzy (pg_trgm similarity, поріг налаштовуваний)
- [ ] Юніт-тести на 30+ реальних прикладах description (з датасету subF-3)

### Seed-довідник
- [ ] Наповнити merchants (is_seed=true): глобальні (Netflix, Spotify, YouTube Premium, iCloud/Apple, Google One, Disney+, ChatGPT, PSN/Xbox...) + українські (Megogo, Sweet TV, Київстар TV, Setanta, оператори, популярні спортзали) — патерни назв, MCC, лого, лінк/інструкція скасування, типові тарифи (для впізнання з 1–2 списань)

### Детекція каденсу
- [ ] SQL: per (user, canonical_key) часовий ряд → інтервали через lag(time) → кластеризація: тижневий (7±2), місячний (28–32), річний (350–380)
- [ ] Стабільність суми: коеф. варіації; окрема гілка «стабільний інтервал + плаваюча сума» (комуналка-стиль) — детектимо, але нижча впевненість
- [ ] Скоринг: f(кількість повторів, регулярність інтервалів, стабільність суми, seed-збіг, MCC-клас) → confidence 0..1; пороги: ≥0.8 auto-detected, 0.5–0.8 «підтверди», <0.5 ігнор
- [ ] Агрегати Apple/Google → status='container' (без розшифровки в MVP)
- [ ] Річні: seed-збіг + ключові слова + одиночне велике списання відомого мерчанта → low-confidence «підтверди»

### Життєвий цикл
- [ ] next_charge_at = last_charge + каденс; перерахунок на кожне нове списання
- [ ] Подія price_increase якщо сума > попередньої (запис у subscription_events)
- [ ] Інкрементальний режим: нова транзакція оновлює тільки свій merchant-ряд (не повний перерахунок)
- [ ] confirm/reject від юзера → статус + detection_feedback; rejected мерчант більше не пропонується цьому юзеру

## Acceptance Criteria / 🚦 ГЕЙТ A
На власній виписці: всі відомі тобі підписки знайдені, хибних спрацювань у high-confidence нуль; інкрементальний прогін на вебхук-транзакції ≤1 сек; повний перерахунок юзера ≤10 сек. Це go/no-go A перед вкладанням у датасет-скейл і UI.
