# subF-12 · Офлайн-оцінка якості детекції — 🚦 ГЕЙТ B перед Фазою 3
**Фаза:** 2 · **Розмір:** S · **Блокери:** subF-3, subF-11

## Задачі
- [ ] Скрипт: прогін рушія (`packages/detection`) по датасету subF-3 → порівняння з ground truth → precision/recall per confidence-tier, звіт у markdown
- [ ] Розбір усіх false positives / false negatives; ітерації нормалізації/скорингу/seed
- [ ] Зафіксувати: скільки «забутих» підписок рушій знайшов би реальним людям (метрика «ага»-моменту)

## GATE B (критерій проходу у Фазу 3)
- precision ≥90% на high-confidence (≥0.8)
- ≥1 «забута» підписка знайдена мінімум у половини людей з датасету

## Якщо gate не пройдено
- precision низька → ітерувати рушій (це нормально, 2–3 кола закладено)
- «забутих» нема ні в кого → перевірка R9: 10 коротких інтерв'ю; можливий півот меседжу в «попередження про списання/подорожчання» до побудови UI
