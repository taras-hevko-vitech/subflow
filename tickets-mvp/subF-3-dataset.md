# subF-3 · Датасет виписок для оцінки детекції
**Фаза:** 0 · **Розмір:** S · **Блокери:** немає

## Мета
Зібрати реальні дані, на яких буде міряти якість рушія (subF-12). Без цього детекція «літає наосліп».

## Задачі
- [ ] Експортувати власну виписку mono за 12 міс (через персональний токен + скрипт, або CSV з застосунку)
- [ ] Попросити 4–9 друзів (за явною згодою) — знеособлені виписки (без ПІБ/номерів карток)
- [ ] Розмітити вручну: для кожної виписки — список реальних підписок (мерчант, каденс, сума), окремо позначити «забуті» (людина не пам'ятала)
- [ ] Формат розмітки: JSON/CSV поряд із випискою; покласти у приватний репо/сховище (НЕ публічний git)

## Acceptance Criteria
≥5 розмічених виписок за ≥6 міс кожна; ground truth по підписках; зафіксовано скільки «забутих» знайшлося при розмітці (перший сигнал R9 — чи існує «ага»-момент).
