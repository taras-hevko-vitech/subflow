# subF-10 · Вебхуки mono + watchdog
**Фаза:** 1 · **Розмір:** S/M · **Блокери:** subF-8

## Задачі
- [ ] `GET /webhooks/mono` → строго HTTP 200 (валідація адреси банком)
- [ ] `POST /webhooks/mono` → миттєве 200, payload {type:"StatementItem", data:{account, statementItem}} → у чергу pg-boss (обробка асинхронна: upsert tx → інкрементальна детекція subF-11 → перевірка «завтра списання» для subF-16)
- [ ] Реєстрація вебхука per-connection після підключення (URL на subflow.app через ALB, з ідентифікатором конекшена або мапінг по account id)
- [ ] Watchdog (щоденна джоба): якщо по активному конекшену немає подій N днів → контрольний statement за останню добу → якщо є пропущені транзакції, вважати вебхук мертвим → ре-реєстрація + добір пропущеного + алерт у Sentry/TG
- [ ] Захист ендпоінта: валідація формату, ліміт розміру, rate limit по IP

## Acceptance Criteria
Нова транзакція з'являється у БД ≤30 сек після реального списання; симуляція «вебхук вимкнено» (3 фейли) → watchdog відновлює без втрати транзакцій.

## Нотатка
mono ретраїть через 60 і 600 сек, після 3-ї невдачі вимикає вебхук — тому відповідь має бути миттєвою (жодної синхронної обробки).
