# subF-17 · Сервісні екрани: профіль, безпека, фідбек
**Фаза:** 3 · **Розмір:** S · **Блокери:** subF-13

## Задачі
- [ ] Профіль: email, статус підключення mono (active/revoked + дата останньої синхронізації), кнопки «Перепідключити» / «Від'єднати банк»
- [ ] «Безпека» (статичний екран): що зберігаємо/не зберігаємо, шифрування, як відкликати токен у mono, лінк на політику приватності
- [ ] Видалення акаунта: подвійне підтвердження → DELETE /me → локальний логаут; чесний текст «видаляємо все і назавжди»
- [ ] Фідбек: «Детекція помилилась?» — форма в один екран (текст + автоконтекст) → detection_feedback
- [ ] Політика приватності + згода на обробку персональних даних (лінк на веб-сторінку або вбудований markdown); дисклеймер «сервіс надає аналітику і не є надавачем платіжних послуг»

## Acceptance Criteria
Видалення акаунта реально чистить БД (тест); всі юридичні тексти на місці до зовнішнього TestFlight (вимога Apple review теж).
